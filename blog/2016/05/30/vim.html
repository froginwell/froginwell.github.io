第一部分 无插件使用vim

基本上大家装上vim后就会自己进行一些设置，为了体验原生vim的威力，可以通过如下命令来启动原始的vim

也可以用下面的命令指定配置文件启动vim，而不加载自己的vimrc

1. 常用操作

命令 作用
h 左
j 下
k 上
l 右
0 将光标移到当前行开头
^ 将光标移到当前行第一个非空字符
$ 将光标移到当前行结尾
gj，gk，g0，g^，g$ 意思不变，只是针对可视行操作
w 将光标移到下一个word开头
b 将光标移到当前word开头或上一个word开头
e 将光标移到当前word结尾或下一个word结尾
ge 将光标移到上一个word的结尾
W，B，E，gE 对WORD进行操作，其它意思同上，word与WORL的区别见示例1-1
f{char} 正向移到当前行的第一个char（;移到下一个 ,移到上一个）
F{char} 反向移到当前行的第一个char（;移到下一个 ,移到上一个）
t{char} 正向移到当前行的第一个char前的字符（;移到下一个 ,移到上一个），有了f为何要t呢？见示例1-2
T{char} 反向移到当前行的第一个char前的字符（;移到下一个 ,移到上一个）
% 跳转到匹配的()或{}或[]
>> 当前行向右缩进
>G 同时向右缩进当前行到尾行的所有行
C 相当于c$，即删除当前字符到行尾的字符（包括当前字符），并进入插入模式
S 删除当前行并进入插入模式
m[a-zA-Z] 在当前光标做标记，小写表示局部，大写表示全局
`[a-zA-Z] 跳到标记处
‘[a-zA-Z] 跳到标记所在行的第一个非空字符
`` 跳转到最后一次跳转之前的位置，它的应用见示例1-3
`. 跳转到最后一次改变的位置
`^ 跳转到最后一次插入的位置
gi 跳转到最后一次插入的位置并进入插入模式
`[ 跳转到最后一次改变或复制的开头
`] 跳转到最后一次改变或复制的结尾
g; 在changes列表回退，可以用:changes查看changes列表
g, 在changes列表前进
<C-o> 根据jumps列表回退，jumps列表的说明见示例1-4
<C-i> 根据jumps列表前进
* 查找当前光标下的单词
gUU 将当前行变为大写
guu 将当前行变为小写
g~~ 翻转当前行大小写
R 进入替换模式，将Tab视为一个空格
gR 与R的不同之处是将Tab视为多个空格
r 替换当前的字母或汉字后，会自动回到正常模式
gr 与r的不同之处是将Tab视为多个空格
gv 重新选中选区
o 将光标移到高亮选区的另一端
q: 打开命令行窗口
zz 将当前光标所在行移到屏幕中间
zt 将当前光标所在行移到屏幕顶部
zb 将当前光标所在行移到屏幕底部
u 在normal模式为撤销，在可视模式为将选区内容变为小写
<C-r> 在normal模式为重做
U 在normal模式为撤销本行的所有操作，在可视模式为将选区内容变为大写
:x 保存（如果有改动的话）并退出
L 移动光标到当前窗口的最后一个非空行
M 移动光标到当前窗口的非空行的中间
H 移到屏幕的第一行
( 跳转到到句子的开头
) 跳转到句子的结尾
{ 跳转动到上一个段落的开头
} 跳转到下一段落的开头
+ 跳转到下一行的第一个非空字符
- 跳转到上一行的第一个非空字符

示例1-1：word与WORD的区别，引自vim

A word consists of a sequence of letters, digits and underscores, or a sequence of other non-blank characters, separated with white space (spaces,tabs, <EOL>). This can be changed with the ‘iskeyword’ option. An empty line is also considered to be a word.

A WORD consists of a sequence of non-blank characters, separated with white space. An empty line is also considered to be a WORD.

示例1-2：有了f为何要t

假如我们有个文件，它的内容如下

int test(int id,char *abcde)

现在光标在char *abcde的第二个a上，我们想把abcde改为name，可以用 ct)，然后输入name，如果我们用 cf)，就会删掉)。当然这个例子最简单的是用 ce，然后输入name。

示例1-3：``的应用

假如我们要把 (cat dog) 的圆括号改为方括号，当前光标在(上，可以先用%跳到)，再用r]将)替换为]，然后用``跳到(，最后用r[将(替换为[。

示例1-4：jumps列表的说明

我们可以用:jumps命令查看当前的jumps列表，一般来说，下列命令所产生的跳转会被记录到jumps列表

命令 作用
[count]G 跳到第count行
使用/pattern或?pattern后的n(N) 跳转到下一个(上一个)匹配项
% 跳转到匹配的()或{}或[]
(和) 跳转到到句子的开头、结尾
{和} 跳转到前一个段落的开头、下一个段落的开头
H和M和L 跳转到屏幕的顶部、中部、底部
gf 跳转到光标下的词所代表的文件
<C-]> 跳转到函数定义
`{mark}和’{mark} 跳转到标记

2. 动作与模式

2.1 动作

动作 意义
d 表示删除
c 表示替换，即删除后进入插入模式
y 表示复制
v 表示选择
g~ 表示翻转大小写
gU 表示变为大写
gu 表示变为小写
> 表示向右缩进
< 表示向左缩进
= 表示自动缩进

2.2 模式

模式 意义
i(或i) 圆括号之间，不包括圆括号
a(或a) 圆括号之间，包括圆括号
‘, “, `, [, ], {, }, <, > 用法同(
it 表示两个标签之间（不包含标签）
at 表示两个标签之间（包含标签）
iw 当前word
aw 当前word加一个空格
iW 当前WORD
aW 当前WORD加一个空格
is 当前句子
as 当前句子加一个空格
ip 当前段落
ap 当前段落加一个空行

3. 插入模式的快捷操作

快捷键 意义
<C-h> 相当于backspace
<C-w> 回删一个单词
<C-u> 回删到行首
<C-o> 回到正常模式，执行一个命令后自动回到插入模式
<C-r>{register} 插入指定寄存器里面的内容
<C-r>={expression}<CR> 插入表达式的结果

<C-o>的一个应用场景为：当你编辑满一个屏幕时，可以按<C-o>，然后执行zz，这样尾行移到屏幕中间后会自动进入插入模式。

4. 命令行模式

操作 意义
:[range]delete[x] 删除特定行到x寄存器，delete可以简写为d
:[range]yank[x] 复制特定行到x寄存器，yank可以简写为y
:[line]put[x] 将x寄存器的内容放在指定行后
:[range]copy{address} 将[range]指定的行复制到{address}指定的行后，copy可以简写为t
:[range]move{address} 将[range]指定的行移动到{address}指定的行后，move可以简写为m
:[range]join 合并指定的行，即去掉换行符
:[range]normal{commands} 在[range]指定的所有行上执行{commands}
:[range]substitute/{pattern}/{string}/[flags] 替换，substitute可简写为s，可以用 :&& 在当前行重复上个替换命令，用 g& 在整个文件重复上个替换命令
:[range]global/{pattern}/[cmd] 在{pattern}匹配并且[range]范围内的所有行上执行[cmd]命令，global可简写为g，如果要在所有非匹配的行上执行命令用g!或vg或v，此命令针对行进行操作
@: 重复最后一条命令
<C-d> 补全命令
<C-r><C-w> 将光标下的word插入到命令行
<C-r><C-a> 将光标下的WORD插入到命令行
:shell 打开一个shell，可以在里面执行命令，用exit退出，退出后会回到vim
!{cmd} 执行shell命令{cmd}
r[ead]!{cmd} 将{cmd}命令的执行结果插入到当前光标所在行后面
[range]w[rite]!{cmd} 用[range]指定的内容作为标准输入来执行{cmd}
[range]!{filter} [range]作为标准输入传给{filter}，然后执行后的结果替换[range]
q: 打开具有命令历史的命令行窗口
q\ 打开具有搜索历史的命令行窗口
<C-f> 从命令行模式转到命令行窗口

关于range的说明

符号 代表的地址
n 第n行
$ 最后一行
0 虚拟行0行
. 光标所在的行
‘m 标记m所在的行
‘< 可视选区的开始
‘> 可视选区的结束
% 整个文件

0的一个应用场景为：如果你想把某些内容复制到（:t）或移动到（:m）第一行之前，可以用0来表示address。下面是一些例子

例子 说明
:t0 将当前行copy到第一行上面，也可以写为:.t0，由于默认地址为当前行，所以可以省略.
:1,4t6 将1到4行的内容复制到第6行之后
:$-2,$t0 将倒数三行的内容复制到第一行之前
:’<,’>t1 将选区内容复制到第一行之后

需要注意的是:t和:m的操作是针对行的操作，也就是说不管你的选区是行选区还是块选区，都会将选区所包含的行复制或移动，并不会针对行内的某些选中元素操作。

5. 文件操作

命令 意义
:ls 列出打开的文件
:bp 上一个文件
:bn 下一个文件
:b{num} 第num个文件
<C-6> 按顺序循环切换文件
:bf 第一个文件
:bl 最后一个文件
:bdelete N1 N2 N3 … 删除指定buffer
N,M bdelete 删除指定范围buffer
<C-w>s 水平分割窗口
<C-w>v 垂直分割窗口
:sp[ilt] 水平分割窗口
:vsp[ilt] 垂直分割窗口
:sp[ilt] {filename} 水平分割窗口并载入文件
:vsp[ilt] {filename} 垂直分割窗口并载入文件
<C-w>w 在窗口之间循环切换
<C-w>c 关闭活动窗口
:cl[ose] 关闭活动窗口
<C-w>o 只留下活动窗口，关闭其他窗口
:on[ly] 只留下活动窗口，关闭其他窗口
<C-w>= 尽可能使所有窗口的高度和宽度相等
<C-w>_ 使活动窗口的高度最大化
<C-w>| 使活动窗口的宽度最大化
[N]<C-w>_ 设置活动窗口的高度为N行
[N]<C-w>| 设置活动窗口的宽度为N列
<C-g> 显示当前文件的状态信息

6. 寄存器

可以用 “{register} 的方式使用寄存器，可以用 :reg {register} 的方式查看某个寄存器的内容，用 :reg 查看所有寄存器的内容。

寄存器名 说明
“ 无名寄存器，x、s、d{motion}、c{motion}、y{motion}的内容会被放到这个寄存器
0 y{motion}的内容除了被放在”寄存器外还会被放在0寄存器
1-9 最近9次删除的内容，这些内容必须包含换行符，否则不会被保存
- 行内删除的内容
_ 黑洞寄存器，当用此寄存器时复制或删除的内容都不会被保存
+ 系统剪贴板寄存器
= 表达式寄存器
% 当前文件的名字
# 当前交替文件名
. 最后一次插入的文本
: 最后一次执行的命令行命令，即 :{cmd} 命令
/ 最后一次的搜索规则，即 /{pattern}
a-z 供用户使用
A-Z 将内容附加到a-z的后面

数字寄存器，也就是0-9。 如果不指定寄存器的名字，那么删除的内容，vim默认是存到1，复制内容是存到0号寄存器。 如果继续删除，那么原来1的内容就转到2，依次类推，当删除超过9的时候， 原先的 8号数字寄存器就转到9， 原先9的数字寄存器内容就会丢失。

7. 宏

开始宏录制用 q{register} ，需要结束按 q ，执行录制的内容用 @{register} ，执行最近的宏命令用 @@ 。

8. 正则表达式

符号 说明
\c 忽略大小写
\C 大小写敏感
\v 使字符具有特殊意义，\v的使用见 示例8-1
\V 消除所有字符的特殊含义
<> 精确的匹配单词，比如 /<int> 会匹配 int，但不会匹配 printf
% 不将它后面的内容放入寄存器，使用方法见 示例8-2
\zs \ze 开始边界与结束边界，使用方法见 示例8-3
